# Sanbrain Scheduling (launchd)

Scheduling runs on **launchd, not cron**. Install/refresh with:

```
bash ~/sanbrain/scripts/install-launchd.sh
```

Check the whole loop's health with:

```
bash ~/sanbrain/scripts/brief-status.sh
```

## Jobs

| Job | When | What |
|------|--------|------|
| `com.sanbrain.nightly` | 22:00 daily | sensors → staged skill chain (4 separate `claude -p` calls) |
| `com.sanbrain.morning` | 07:00 daily | brief stylesheet install → vault-doctor → morning brief (auto-opens in Obsidian) |
| `com.sanbrain.brief-watcher` | seconds after any `wiki/daily/` change, + 30-min backstop | transcribe voice recordings embedded in the brief + propagate written edits. mtime-gated: fires are free unless the brief actually changed |

Why launchd, not cron:
- **WatchPaths** fires the feedback processor within seconds of Santiago saving
  an edit or stopping a recording — cron polling would add minutes of latency.
- **StartCalendarInterval** fires missed jobs on wake; cron silently skips
  runs while the Mac is asleep.

The plists in `~/Library/LaunchAgents/com.sanbrain.*` are GENERATED by
`install-launchd.sh` — never hand-edit them; change the installer and re-run.

## macOS permissions (TCC) — read this before debugging

Tested 2026-06-10 with one-shot launchd jobs:

- Under launchd, `/bin/bash` and `python3` get **"Operation not permitted"**
  reading vault file CONTENT (the Obsidian iCloud container). `stat`/metadata
  works. The `claude` binary and its children ARE allowed.
- Therefore the watcher wrapper only stat-gates and notifies; all content work
  (reading the brief, Whisper transcription via `transcribe-brief-audio.sh`,
  propagation) runs INSIDE the claude call.
- Optional cleanup: granting Full Disk Access to `/bin/bash` (System Settings →
  Privacy & Security) removes the remaining "Operation not permitted" noise and
  is REQUIRED for vault-git.sh / vault-doctor.py to work under launchd.

## The feedback loop (what Santiago experiences)

```
07:00  brief generated, auto-opens in Obsidian (desktop)
       Santiago reads; hits ⌘⇧R and talks while reading, and/or checks boxes
~10s   watcher fires on save → stat gate passes
~30s   🎙️ Telegram: "grabación recibida y transcrita (N palabras), procesando…"
~3-5m  claude propagates, checks Q boxes with his answers, appends ## Respuestas
       📝 Telegram: summary of what was answered and propagated
       (on failure: ⚠️ Telegram with the log path — silence always means
        "nothing happened", never "something broke quietly")
```

State lives in `~/sanbrain/.state/` (transcript cache, processed list, attempt
counters, heartbeats, last summary). Logs in `~/sanbrain/logs/`.

## Prerequisites
- `claude login` run at least once in Terminal.app (OAuth in macOS Keychain)
- Secrets in `~/.sanbrain.env` (chmod 600) — `OPENAI_API_KEY` for Whisper.
  Cron/launchd do not read `.zshrc`.
- `ffmpeg` (brew install ffmpeg) for chunking voice recordings >25MB (nightly
  meetings harvest; brief feedback recordings are capped at 25MB)
- Obsidian audio recorder + brief stylesheet: installed automatically by
  morning.sh (`scripts/install-brief-style.sh`)

## Reliability machinery (Phase 1)

- **Vault versioning**: `scripts/vault-git.sh` keeps the vault in a git repo whose
  git dir lives OUTSIDE iCloud (`~/.sanbrain-vault.git`). Checkpoints: pre-nightly,
  post-sensors, after every skill stage, post-morning-brief, post-brief-feedback.
  Undo a bad night: `scripts/vault-git.sh log` then `git --git-dir=~/.sanbrain-vault.git revert <sha>`.
  Checkpoint refuses mass deletions (>20% tracked files) — the iCloud-eviction signature.
- **State checkpoints**: `~/sanbrain/.state/<skill>.last` — machine-written ISO
  timestamps. Skills read these; only wrapper scripts write them.
- **Handoffs**: each nightly stage writes `~/sanbrain/.state/handoffs/<skill>.json`;
  the next stage receives it in its prompt. Stages with provably nothing to do
  are skipped mechanically (empty raw/, no new Claude sessions).
- **Heartbeats**: every sensor and stage writes `~/sanbrain/.state/heartbeats/<name>.json`.
  The brief-watcher heartbeats on EVERY poll (ok/skip/error), so a dead watcher
  is distinguishable from an idle one. `vault-doctor.py` surfaces any sensor
  with no success in 48h into the brief.
- **Alerts**: failures push to Telegram (San group, sanbrain topic) via openclaw.
  Persistent error conditions alert once per day (`notify_once`), not per poll.
- **vault-doctor**: `scripts/vault-doctor.py` runs before each brief — broken
  wikilinks, iCloud conflict/evicted files, schema violations, unpropagated
  meeting attendees, raw/ backlog, sensor health. Report: `wiki/logs/doctor-DATE.md`.

## Data Flow

```
Sensors (6 input channels):
  ~/Downloads/            → process-downloads.py (lifecycle) + harvest-downloads.sh → raw/downloads-manifest-*.md (system file, read-only report)
  GitHub API (2 repos)    → harvest-github.sh     → raw/github-prs-*.md
  iCloud Meetings/ (.m4a) → harvest-recordings.sh → raw/voice-*.md (Whisper, chunked >25MB; 📱 Telegram ping per memo)
  OpenClaw conversations  → harvest-openclaw.sh   → raw/openclaw-conversations-*.md
  Claude Code sessions    → ~/.claude/projects/   (read directly by claude-extract)
  Calendar.app (4 cals)   → schedule-reminders.sh → raw/today-reminders-*.md + Telegram agenda push

                        ↓
21:00 downloads-triage.sh (launchd, claude -p)
  Advisory LLM pass over ~/Downloads + the vault. Writes ONLY an advisory
  proposal (.state/downloads-proposal.json): per-file keep/disposable/save-
  review/save-legal/crypto. Touches no files. (claude can read the vault to ask
  "already captured?" — the deterministic executor can't.)
                        ↓
22:00 nightly.sh (launchd)
  Phase 0: vault-git checkpoint (undo point)
  Phase 1: process-downloads.py — Downloads = working directory. Consumes the
           21:00 proposal CLAMPED (the LLM can only make a file safer — never
           downgrade crypto/legal, never delete). Trashes only two-way doors
           (already-saved-in-vault OR idle ≥14d), saves unsaved valuables into
           raw/legal | raw/needs-review FIRST, never touches crypto keys. All
           moves go to macOS Trash (recoverable ~30d), never os.remove. Audit
           trail: wiki/logs/downloads-trash-YYYY-MM.md.
  Phase 2: 4 harvest sensors (each heartbeats; failures alert, don't halt)
  Phase 3: staged skill chain — one claude -p call PER skill:
    1. ingest:           raw/ → wiki/ → raw/archive/   (skipped if raw/ empty)
    2. claude-extract:   ~/.claude/ → wiki/            (skipped if no new sessions)
    3. entity-update:    back-links, stubs, enrichment
    4. context-maintain: rewrite context files
  Phase 4: archive stale system files, sentinel, Telegram alert if failures
                        ↓
07:00 morning.sh (launchd)
  5. install-brief-style → CSS snippet + audio recorder into .obsidian/
  6. schedule-reminders  → calendar events + agenda push to Telegram
  7. vault-doctor        → wiki/logs/doctor-DATE.md (mechanical health)
  8. morning-brief       → wiki/daily/YYYY-MM-DD-brief.md (uses doctor numbers)
                        ↓
Santiago reads the brief on his desktop in Obsidian. Feedback two ways:
  - Voice: ⌘⇧R anywhere (skhd → scripts/voice-memo) — press to record, press to
    finish. Recording → CloudDocs/Recordings/ → immediate Whisper → raw/ +
    queued for the brief processor. (Obsidian's embed-in-note recorder also
    works, via command palette.)
  - Written: checks boxes, writes answers inline
                        ↓
Seconds later: brief-watcher fires (launchd WatchPaths; 30-min backstop)
  → claude run: transcribe-brief-audio.sh (Whisper, cached, attempt-capped,
    🎙️ ping on transcription) → propagate answers/decisions to entities +
    context → check Q boxes, append ## Respuestas to the brief
  → 📝 Telegram summary of what was done; ⚠️ on failure. Never silent failure.
```
