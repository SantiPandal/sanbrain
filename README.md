# Sanbrain

A personal second brain that compounds knowledge over time. Cron jobs prompt an AI harness to run skills that capture, connect, and compile knowledge into an Obsidian vault.

Inspired by [GBrain](https://github.com/garrytan/gbrain). Stripped to the minimum: cron jobs, thin prompts, skill files, and an Obsidian vault as the database.

## Architecture

```
crontab (Mac mini, always-on)
  | thin prompts
  v
Harness (OpenClaw / Claude CLI / any LLM agent)
  | reads skill, executes phases
  v
Obsidian Vault (iCloud) = THE BRAIN
  | syncs
  v
Phone (read on mobile)

Claude Code (session-based, manual work)
  | reads context file generated by harness
  v
Coding, analysis, manual tasks
```

## How It Works

1. **Cron fires** a thin prompt: `"Read skills/daily-compile/SKILL.md and run it."`
2. **Harness reads** the skill file (markdown with phases, contracts, output format)
3. **Harness executes** — reads from vault, processes data, writes pages back to vault
4. **Entity propagation** auto-updates all related people/business pages
5. **Knowledge compounds** — every interaction enriches the next one

## The Skills

| Skill | What It Does | Trigger |
|-------|-------------|---------|
| `claude-extract` | Pull insights from Claude sessions into vault | Cron: every 30 min |
| `meeting-ingest` | Process meeting transcripts → structured pages + entity propagation | Manual / on new file |
| `media-ingest` | YouTube, X, articles → source material pages | Manual / queued |
| `book-mirror` | Chapter-by-chapter personal mirror of a book | Manual |
| `entity-propagate` | After any write, update all mentioned people/business pages | Post-write hook |
| `enrich` | Upgrade people pages from stub → basic → full based on mention count | Threshold trigger |
| `daily-compile` | Nightly dream cycle: synthesize, connect, update stale pages | Cron: 11 PM |
| `context-for-claude-code` | Generate context file for Claude Code sessions | Cron: 6 AM |

## Quick Start

```bash
git clone https://github.com/santipandal/sanbrain.git
cd sanbrain

# 1. Create vault folders (point to your Obsidian vault)
chmod +x setup/create-vault-folders.sh
./setup/create-vault-folders.sh /path/to/your/vault

# 2. Write your identity files (see setup/seed-identity.md)

# 3. Install cron jobs
crontab crontab.example  # edit paths first

# 4. Configure your harness (OpenClaw, Claude CLI, etc.)
```

## Page Schema

Every page follows **compiled truth + timeline**:
- **Above the line**: current best understanding (rewritten when evidence changes)
- **Below the line**: append-only timeline (never edited, only added to)

See `templates/` for page formats and `examples/` for filled examples.

## Conventions

- `skills/conventions/quality.md` — back-linking, citations, notability gate
- `skills/conventions/vault-schema.md` — compiled truth + timeline format
- `skills/conventions/entity-rules.md` — tiered enrichment system

## Philosophy

Fat skills. Thin harness. The LLM is the engine, not the car. The vault is the brain, not a filing cabinet. Cron jobs are the heartbeat. Everything compounds.
