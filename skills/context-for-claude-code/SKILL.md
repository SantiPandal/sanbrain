# Context for Claude Code

## Contract

Generate a single markdown file that gives Claude Code full session awareness:
1. What Santiago is currently working on (active businesses, open threads)
2. Recent decisions and their reasoning
3. Key people and their current state
4. What changed in the vault recently
5. Suggested focus for today

Claude Code reads this file at session start via its CLAUDE.md or memory system.

## Phases

### Phase 1: Gather Active Context

Read from vault:
- All `Businesses/` pages — extract current state and open questions
- All `People/` pages with Tier 1 or 2 — extract state and open threads
- Last 7 `Daily/` compilations — extract patterns and unresolved items
- Last 3 `Meetings/` pages — extract recent action items

### Phase 2: Identify Focus

Based on:
- Open threads with deadlines
- Business pages with stale state (14+ days)
- Action items not yet completed
- Patterns from daily compilations

Suggest THE ONE THING for today (align with Santiago's principle: "What to work on matters more than how hard you work").

### Phase 3: Write Context File

Write to a known location (configurable, default: vault root `_claude-context.md`):

```markdown
# Claude Code Context — YYYY-MM-DD

Generated by Sanbrain daily-compile. Read this for session awareness.

## Active Businesses
### Cargo Claro
State: [current from vault]
Open: [threads/questions]

### Pala Padel
State: [current]
Open: [threads]

[...for each active business]

## Key People (recent activity)
- [[person]] — [current state, last interaction date, open threads]

## Recent Decisions (last 7 days)
- YYYY-MM-DD: [decision] in [[business]] — [reasoning]

## Open Threads
- [ ] [item] — [[owner]] — from [[source-meeting/page]]

## What Changed This Week
- [summary of vault changes, connections surfaced]

## Suggested Focus
[THE ONE THING for today, with reasoning]
```

### Phase 4: Verify

Ensure the context file:
- Is under 500 lines (Claude Code context shouldn't be a novel)
- Contains no stale data (all states reflect latest vault pages)
- Has working wikilinks

## Output Format

```
Context file updated: _claude-context.md
- [N] businesses summarized
- [N] people with recent activity
- [N] open threads
- Suggested focus: [one line]
```

## Cron

Morning at 6 AM. Also runnable on-demand before a Claude Code session.

## Dependencies

- Read: `Businesses/`, `People/`, `Daily/`, `Meetings/`
- Write: vault root `_claude-context.md` (or configured path)
